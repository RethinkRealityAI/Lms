## Pre-Audit State

All three tables had RLS enabled (`relrowsecurity = true`). No policies used the inline `FROM users` pattern that causes infinite recursion.

### `slides` ‚Äî policies found

| Policy | Command | Notes |

| Students can view published slides for enrolled courses | SELECT | Checks `status = 'published'`, `deleted_at IS NULL`, and enrollment via `course_enrollments`. Correct pattern. |

### `slide_templates` ‚Äî policies found

|---|---|---|

| Anyone can view global templates | SELECT | `institution_id IS NULL`. Correct. |

### `content_activity_log` ‚Äî policies found

| Policy | Command | Notes |
|---|---|---|
| Admins can view own institution activity | SELECT | Uses `is_admin()` + `current_institution_id()`. Correct. |
| Admins can insert activity for own institution | INSERT | **WRONG** ‚Äî restricted to admins only; client-side `logActivity` calls are made by non-admin authenticated users. |

## Changes Made (migration 014_fix_rls_audit.sql)

### 1. `content_activity_log` INSERT policy ‚Äî FIXED

**Before:** `Admins can insert activity for own institution` ‚Äî required `is_admin()`, blocking non-admin users from logging activity.

**After:** `Authenticated users can log activity` ‚Äî allows any authenticated user whose `institution_id` matches `current_institution_id()` to insert. This is the correct pattern for client-side activity logging.

```sql
DROP POLICY IF EXISTS "Admins can insert activity for own institution" ON public.content_activity_log;

CREATE POLICY "Authenticated users can log activity" ON public.content_activity_log
  FOR INSERT WITH CHECK (
    auth.role() = 'authenticated'
    AND institution_id = public.current_institution_id()
  );
```

### 2. `slide_templates` SELECT policy ‚Äî ADDED

**Before:** Only institution-scoped SELECT policies existed (global templates and own-institution templates as separate rules).

**After:** Added explicit `Authenticated users can read templates` policy covering all authenticated users. This ensures the template picker UI works correctly for all roles without relying on institution scope conditions.

```sql
CREATE POLICY "Authenticated users can read templates" ON public.slide_templates
  FOR SELECT USING (auth.role() = 'authenticated');
```
